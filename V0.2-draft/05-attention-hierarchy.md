# 上下文编排器 / Context Orchestrator

> 四层注意力位阶 + 片段编码规范
>
> 📝 协议设计，尚未完整实现。当前仅实现热层（conversation_history）+ 简化前置区（已 confirmed 文件内容）。

## 四层注意力位阶

```
┌─────────────────────────────────────────┐
│  位阶1：系统提示词区                     │
│  · 认知契约（禁止条款+工作流）            │
│  · 书童身份定义                          │
│  · 固定不变，每次对话完整注入              │
├─────────────────────────────────────────┤
│  位阶2：前置区（热层）                    │
│  · 当前会话已确认事实列表                  │
│  · .shutong/ 下已confirmed的文件内容      │
│  · 只追加，不删除，不修改                  │
├─────────────────────────────────────────┤
│  位阶3：中置区（温层/冷层）               │
│  · 相关记忆片段（按相关性排序）             │
│  · 项目级业务规则                         │
│  · 技能库引用                            │
│  · 每条带元数据：来源、时效、信任度         │
├─────────────────────────────────────────┤
│  位阶4：近因区                            │
│  · 用户当前输入（原文）                    │
│  · 上一轮对话历史（最近N轮）               │
└─────────────────────────────────────────┘
```

## 片段编码规范

所有进入上下文的记忆片段必须编码为结构化文本：

```json
{
  "fragment_id": "frag_001",
  "content": "用户确认使用JWT进行身份验证",
  "source": "session_003_round_002",
  "source_type": "user_confirmed",
  "timestamp": "2026-07-19T10:30:00Z",
  "topic": "authentication",
  "trust_level": "absolute",
  "expiry": null
}
```

## 编排规则（确定性）

1. 前置区片段：source_type="user_confirmed" 的按时间序全量注入
2. 中置区片段：按 topic 相关性召回，按 trust_level 排序，时间截断
3. 截断策略：优先截断中置区，其次近因区，**前置区绝不截断**
4. 冲突处理：同一 topic 存在矛盾片段时，全部注入并标注"存在冲突"，由模型在认知显化中暴露
