# 认知状态机 / Cognitive State Machine

> **协议版本**：书童协议 V0.2-draft  
> **日期**：2026-07-20  
> **定位**：书童协议的核心运行时  
> **核心命题**：状态切换由模型语义判断决定，不是系统硬编码

---

## 一、一句话定位

**书童协议是一个认知状态机。**

状态切换不由系统规则触发，由模型语义判断触发。系统只负责校验状态切换的合法性。

---

## 二、状态定义

### 六大状态

```
                    ┌─────────────┐
                    │    START    │
                    │  接收用户输入 │
                    └──────┬──────┘
                           │
                           ▼
                    ┌─────────────┐
                    │   THINK     │ ← 模型判断：当前该思考还是追问？
                    │   思考状态   │
                    └──────┬──────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
        信息充足           │      信息不足/不确定
              │            │            │
              ▼            ▼            ▼
        ┌─────────┐  ┌─────────┐  ┌─────────┐
        │ ANSWER  │  │ EXECUTE │  │  QUERY  │
        │ 回答状态 │  │ 执行状态 │  │ 追问状态 │
        │         │  │         │  │         │
        └────┬────┘  └────┬────┘  └────┬────┘
             │            │           │
             └────────────┴───────────┘
                           │
                           ▼
                    ┌─────────────┐
                    │   LOCKED    │ ← 理解已固化，进入只读
                    │   锁定状态   │
                    └─────────────┘
                           │
              （定期触发）   │
                    ┌──────┴──────┐
                    │   REFLECT   │ ← 后台记忆整理
                    │   反思状态   │
                    └─────────────┘
```

### 状态详解

| 状态 | 触发条件 | 模型行为 | 用户感知 | 产出 |
|------|---------|---------|---------|------|
| **THINK** | 信息充足，可推理 | 内部推理，暴露假设清单 | "AI 在思考" | 认知显化 |
| **QUERY** | 信息不足/不确定 | 暴露盲区，向用户提问 | "AI 在问我" | 追问内容 |
| **ANSWER** | 已确认信息足够 | 基于已确认事实回答 | "AI 回答了" | 确定性回答 |
| **EXECUTE** | 用户确认后 | 调用工具/写入文件 | "AI 在执行" | 执行结果 |
| **LOCKED** | 产品包锁定/任务完成 | 只读，变更需重新对齐 | "已锁定" | 固化记忆 |
| **REFLECT** | 会话结束/定期触发 | 梦境整合、记忆沉淀 | **无感知** | 温层记忆 |

## 三、状态切换机制

### 不由系统硬编码，由模型语义判断

**错误理解（系统编排）：**
```
if 用户输入包含"?" → 进入 QUERY
if 用户说"确认" → 进入 EXECUTE
if 轮次 > 3 → 进入 ANSWER
```

**正确理解（模型嵌入判断）：**
```
用户输入 → 模型语义判断 → "当前信息是否充足？"
  → 充足 → THINK
  → 不足 → QUERY
  → 已确认 → ANSWER
  → 用户说"确认" → EXECUTE
  → 任务完成 → LOCKED
```

**关键：状态切换是模型的认知判断，不是系统的条件触发。**

### 系统只校验合法性

```
模型判断：进入 EXECUTE
  → 规则层校验：是否有用户确认？
    → 有 → 允许切换
    → 无 → 拦截，强制进入 QUERY
```

**规则层不决定切换什么状态，只校验"这个切换是否合法"。**

## 四、THINK vs REFLECT：本质区别

| 维度 | THINK（思考） | REFLECT（反思/梦境整合） |
|------|--------------|------------------------|
| **触发时机** | 对话过程中，实时 | 会话结束后，后台 |
| **处理对象** | **当前任务**——用户刚输入的内容 | **历史记忆**——热层未整理的原始对话 |
| **用户感知** | 有（可能暴露假设清单） | **无**（纯后台静默执行） |
| **产出** | 当前决策（进入 ANSWER 还是 QUERY） | 整理后的温层记忆 |
| **是否打扰用户** | 可能 | **绝不** |
| **类比人脑** | 工作记忆内的实时推理 | 睡眠时的记忆巩固 |

### THINK 场景

```
用户：分析一下这个架构的优缺点
  → 模型判断：信息充足，进入 THINK
  → 内部推理：列出优缺点
  → 暴露假设：「我假设你说的是微服务架构，来源：上下文推断」
  → 用户确认假设 → 进入 ANSWER
```

### REFLECT 场景

```
会话结束（或凌晨3点定时触发）
  → 读取热层 30 轮原始对话
  → 提取已确认事实
  → 去重：「登录方式」出现了3次，合并为一条
  → 消化矛盾：「JWT→Cookie→JWT」合并为「JWT（兼容后）」
  → 写入温层
  → 用户完全无感知
```

**一句话：THINK 是"现在该怎么回应"，REFLECT 是"刚才的对话该怎么整理进长期记忆"。**

## 五、状态与记忆的关系

### 每个状态产出记忆

| 状态 | 产出记忆类型 | 写入层级 |
|------|------------|---------|
| THINK | 认知显化（假设清单、盲区清单） | 热层（临时） |
| QUERY | 追问-确认链 | 热层（临时） |
| ANSWER | 已确认回答 | 热层（临时） |
| EXECUTE | 执行结果 | 热层（临时） |
| LOCKED | 锁定的事实 | 温层（固化） |
| REFLECT | 整理后的记忆 | 温层（固化） |

### 记忆流动

```
热层（原始对话）
  → 30轮或70%上下文 → 触发整理
    → REFLECT 状态（梦境整合）
      → 去重、合并、消化矛盾
        → 写入温层（已对齐的理解）
          → 执行层读取
```

## 六、与现有"思考模式"的关系

### DeepSeek R1 / Claude Extended Thinking

| 维度 | 现有思考模式 | 书童协议的 THINK |
|------|-----------|----------------|
| **本质** | 模型内部链式推理（CoT） | 认知状态之一 |
| **暴露假设** | ❌ 不暴露 | ✅ 必须暴露假设清单 |
| **追问用户** | ❌ 不追问 | ✅ 不确定时切换到 QUERY |
| **用户角色** | 旁观者 | **校准者** |
| **产出** | 内部推理过程 | 认知显化 + 状态决策 |

**书童协议的 THINK 状态 = 现有思考模式 + 认知显化（暴露假设）+ 追问（不确定时问）。**

## 七、核心主张

> **书童协议是一个认知状态机。**
>
> **状态切换不由系统硬编码，由模型语义判断触发。**
>
> **系统只校验切换的合法性，不决定切换什么状态。**
>
> **追问不是机制，是 QUERY 状态——和 THINK、ANSWER、EXECUTE、REFLECT 同级。**
>
> **THINK 面向当下决策，REFLECT 面向历史沉淀。**
