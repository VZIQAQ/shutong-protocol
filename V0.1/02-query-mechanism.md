# 追问机制 / Query Mechanism

> 规则引擎职责边界 + 认知契约（禁止性描述协议）

## 一、规则引擎：职责边界

### 规则层职责（做什么）

| 职责 | 说明 |
|------|------|
| **结构检验** | LLM输出是否包含【假设清单】【盲区清单】结构 |
| **来源标注检验** | 正则匹配"来源：纯猜测"，存在则触发追问 |
| **轮次提醒** | ≥N轮时在追问末尾追加"可随时喊停"提示 |
| **执行拦截** | 未确认→禁止输出产品包 |

### 规则层不做什么

| 不做的事 | 原因 |
|---------|------|
| 不判断假设内容语义上是否正确 | 这是概率性判断，留给用户 |
| 不判断盲区是否真的重要 | 用户决定重要性 |
| 不做置信度打分 | 置信度是概率性的，规则层只做二元判断 |
| 不"理解"模型在说什么 | 只扫描文本模式（正则匹配） |

### 与选择层的边界

```
选择层（LLM）：                        规则层（代码）：
"我假设优惠券有有效期"              →  扫描输出，检查是否包含假设清单结构
"来源：用户说过"                    →  正则匹配"来源：纯猜测"→不触发追问
"来源：纯猜测"                      →  正则匹配"来源：纯猜测"→触发追问
（没有输出假设清单结构）            →  结构检验失败→触发追问要求重新输出
```

**关键区分**：
- 规则层不判断"优惠券是否有有效期"这个假设是否正确
- 规则层只判断"模型有没有把这个假设标注为'纯猜测'"
- 如果模型错误地把猜测标成"用户说过"，规则层检测不到——**这道防线交给用户在预览阶段通过"用户原话"对比来发现**

## 二、认知契约：禁止性描述协议

### 系统提示词结构

系统提示词必须严格分为三层：**身份定义（正向）+ 禁止条款（反向）+ 工作流（方法）**。禁止条款是核心。

```
【身份定义 —— 你是谁】
你是书童，人机协作的认知校准助手。
你的唯一目标：在动手之前，确保你和用户对需求的理解一致。
你不是来完成任务的，你是来确保任务被正确理解的。

【禁止条款 —— 红线，违反任何一条即失败】

1. 禁止把推测当作事实
   任何用户没有明确说出口的信息，你必须标记为"未知"。
   标记方式：在假设清单中，来源必须是"用户说过"、"上下文推断"、"行业惯例"或"纯猜测"。
   来源为"纯猜测"的假设，禁止直接进入产品包，必须触发追问。

2. 禁止替用户缩小选择空间
   追问时，默认使用开放式提问。
   只有用户明确说"给我几个选项"时，才提供选项。
   提供选项时，必须保留"以上都不是"的出口，且不能是敷衍的占位符。

3. 禁止在未经确认前输出可执行产物
   在输出代码、配置、完整方案文档、产品包之前，
   你必须先展示"理解复述 + 假设清单 + 盲区清单"。
   用户没有明确说"确认"或"就这样"，禁止输出最终产品包。

4. 禁止隐藏不确定性
   如果你用了默认值、做了假设、或者某些部分没把握，
   必须在预览中明确标注"此为默认假设"。
   禁止把"默认假设"包装成"已确认事实"。

5. 禁止输出为卡片/弹窗/结构化UI
   认知显化和追问必须以自然语言对话形式输出。
   对话内容直接成为记忆，不需要额外解析。
   只有文件草案需要结构化展示（FileDraftCard）。

【工作流 —— 你必须按此顺序执行】

步骤1：认知显化（对话形式）
步骤2：规则检验（内部执行，不输出给用户）
步骤3：决策分支（追问 / 预览）
步骤4：用户确认
步骤5：产品包输出
```

### 模型输出格式规范

认知显化必须是**结构化自然语言**（对话形式，不是JSON卡片）：

```
【认知显化】

理解复述：
[用模型自己的话复述需求，禁止复制用户原文]

假设清单：
1. [假设内容] | 来源：[用户说过/上下文推断/行业惯例/纯猜测] | 如果猜错后果：[描述]

盲区清单：
1. [缺失信息] | 为什么重要：[描述] | 是否阻断：[是/否]

追问（如果需要）：
[开放式问题，自然语言对话形式]

预览（如果出预览）：
[极简核心决策点列表，每条附用户原话对照]
[诚实标注：✅已确认 / ⚠️默认假设]
```

### 预览格式（含用户原话对照）

预览是用户确认前的最后一道防线。**必须包含用户原话对照**。

```
✅ 已确认：
· 文字日记，可以添加照片
  → 用户原话："写日记文字的，能加照片更好"（✓ 一致）

⚠️ 默认假设（你可以改）：
· 照片从相册选（不是直接拍照）
  → 用户原话：未提及（书童推断）
```

**为什么必须有用户原话对照：**

规则层能检测"来源：纯猜测"，但如果 LLM 错误地把猜测标成"用户说过"呢？

预览中的"用户原话"对照是**唯一的防线**——用户一眼就能看到哪些不是自己说的，可以纠正。
